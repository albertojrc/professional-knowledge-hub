# UX Architecture Sprint Roadmap

## Goal

Turn the Hub into a smoother professional learning platform with a clear structure:

```text
Module → Submodule → Topic → Deep Detail View
```

## Sprint sequence

### Sprint 5.26 — UX Architecture Cleanup

Clean `ModuleCurriculumStudio`, update sidebar copy and make the curriculum architecture the official module rule.

### Sprint 5.27 — Module Detail UX Polish

Improve submodule navigation, topic cards, selected-state behavior, empty states, spacing and responsive behavior.

### Sprint 5.28 — CFA Curriculum Expansion

Create missing CFA foundation assets: Ethics, Quant, Economics, Financial Statement Analysis, Corporate Issuers, Equity, Fixed Income, Derivatives, Alternatives and Portfolio Management.

### Sprint 5.29 — Visual Output Type System

Add a visual output type per asset so each concept displays the right visual: chart, formula card, matrix, checklist, flow, decision table or fallback.

### Sprint 5.30 — Deep Detail Content Cleanup

Review the concept-detail page and remove generic wording. Each block must be concrete, useful and connected to professional application.

### Sprint 5.31 — Route Registry Refactor

Reduce `App.tsx` route complexity by organizing routes into primary, internal, reference and governance registries.

### Sprint 5.32 — Search-to-Curriculum Alignment

Align Global Search results with the curriculum structure so results can open exact topic views or focused submodules.

### Sprint 5.33 — Knowledge Map Curriculum Sync

Align Knowledge Map with modules, submodules, topics, outputs and decision links.

### Sprint 5.34 — UX Consistency Audit

Audit spacing, cards, typography, copy, side panels, empty states, responsive layout and repeated visual patterns.

### Sprint 5.35 — Final Product Polish & Deploy Prep

Final UI polish, validation sequence and deploy readiness review.
