# Sprint 2 Roadmap — Source-Based Knowledge Consolidation

Sprint 2 moves the project from platform foundation to source-based knowledge consolidation.

The objective is to connect the app to the actual material from the user's studies and progressively transform class content into professional knowledge assets.

## Sprint 2 main objective

Convert real academic and professional material from the dual degree into structured, connected, reusable knowledge inside the app.

## Sprint 2 principles

1. Do not invent class coverage.
2. Separate source-based content from recommended complementary content.
3. Mark missing topics as recommended gaps.
4. Consolidate duplicates across courses.
5. Connect concepts to tools, formulas, outputs, cases and decisions.
6. Prioritize banking/data analytics professional relevance.

## Sprint 2 proposed phases

### Sprint 2.1 — Source Inventory System

Create a source inventory layer for files, courses, modules and document types.

Deliverables:

- Source metadata model
- Course/module registry
- Source coverage status
- File-to-topic mapping template

### Sprint 2.2 — Course Area Mapping

Map source material into major professional areas:

- Data Science
- Machine Learning
- Statistics
- SQL / Databases
- Big Data
- Banking
- Finance
- Economics
- Management
- Marketing
- Operations
- Analytics / BI

Deliverables:

- Course-to-area map
- Topic coverage matrix
- Duplicated concept detection list

### Sprint 2.3 — Source-Based Asset Expansion Batch 1

Create the first asset batch from actual course materials.

Priority areas:

- Data Science workflow
- SQL and data modeling
- Statistics foundations
- Machine learning evaluation
- Credit risk and banking analytics

Deliverables:

- 20-30 new or expanded knowledge assets
- Source coverage fields
- Recommended gap labels when needed

### Sprint 2.4 — Output Interpretation Expansion

Expand the Output Atlas using class outputs and professional analytics outputs.

Target outputs:

- Classification report
- Regression summary
- Confusion matrix
- ROC / PR curves
- Calibration plot
- SHAP plots
- Time series forecast output
- Dashboard KPI cards
- SQL query outputs
- Financial ratio tables

### Sprint 2.5 — Formula & Model Library Expansion

Build a richer reference system for formulas and models.

Target coverage:

- Statistics formulas
- Finance formulas
- Risk formulas
- ML model families
- Time series models
- Optimization / operations models

### Sprint 2.6 — Business Case Consolidation

Transform course cases and assignments into professional playbooks.

Examples:

- Credit scoring
- Fraud detection
- AML/KYC
- Customer segmentation
- Marketing campaign analytics
- Financial valuation
- Operations performance
- Macroeconomic scenario analysis

### Sprint 2.7 — Knowledge Graph Upgrade

Create deeper concept dependencies and pathways.

Deliverables:

- Prerequisite map
- Concept dependency graph
- Repeated concept map
- Banking relevance score
- Data analytics relevance score

### Sprint 2.8 — Source Coverage QA

Audit which assets are supported by source material and which are complementary.

Deliverables:

- Source coverage report
- Gap report
- Duplicate consolidation report
- Recommended complementary topic list

## Sprint 2 output target

By the end of Sprint 2, the Hub should contain:

- A source inventory
- A course-to-topic map
- A source-aware Knowledge Library
- Expanded Output Atlas
- Expanded Formula Library
- Expanded Model Library
- Business cases from class material
- Clear missing-topic and complementary-topic labels

## Sprint 2 success criteria

Sprint 2 is successful when a user can answer:

- Which course taught this concept?
- Where does this concept fit professionally?
- What outputs should I know how to interpret?
- What formulas or models are connected?
- What banking or business decisions does it support?
- What topics are missing and should be complemented?

## First recommended Sprint 2 action

Start with Sprint 2.1: Source Inventory System.

Before adding more content, the project needs a clean source registry so every future asset can be traced back to a class, document or recommended complement.
